# M1b — ZERO Frontend Read-only Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the full UI shell (Sidebar + Topbar + AiPanel placeholder) and all 9 read-only screens (Dashboard, Inbox, Test Cases, Test Runs, Defects, Analytics, Traceability, Integrations, Docs) wired to M1a's GET endpoints with capability gating, tier badge, empty/loading/error states, and visual parity >=95% with mockup. AI panel hidden in ZERO tier.

**Architecture:** Vite + React 19 SPA. TanStack Router file-based routing under `apps/web/src/routes/`. TanStack Query for server state with stale-while-revalidate, suspense boundaries, error boundaries. Zustand for global UI state (capabilities, ai panel toggle, command palette). shadcn/ui primitives + Tailwind 4 tokens from UI_SPEC. Capability fetched on app boot, stored in Zustand, used by `<Gated>` wrapper to conditionally render AI features. Lucide icons. Geist Sans + Mono fonts.

**Tech Stack:** Vite 6, React 19, TypeScript 5, TanStack Router (file-based), TanStack Query v5, Zustand 4, Tailwind 4, shadcn/ui, Radix UI, Lucide React, Recharts, dnd-kit (placeholder; used in M1d), Monaco Editor (placeholder; used in M1d), date-fns, clsx + tailwind-merge, axios, vitest, @testing-library/react, msw (mock service worker), storybook 8, geist font.

## Conventions used in this plan

- **TDD-first** for any non-trivial logic (gating decisions, capability resolution, ws reconnect, error mapping). Write the vitest before the implementation. Visual-only components allow snapshot-after.
- **File-based routing**: TanStack Router file path = URL. `routes/_app/dashboard.tsx` → `/dashboard` inside `_app` layout. Underscore prefix = pathless layout segment.
- **No barrel files**: import directly from `@/components/shell/Sidebar` not `@/components/shell`.
- **No `any`**: prefer `unknown` + narrow, or generated OpenAPI types.
- **Commit cadence**: 1 commit per acceptance checkbox group (sub-task). Conventional commits — `feat(web): add Sidebar shell` / `test(web): cover Gated ZERO/CLOUD branches` / `chore(web): wire Tailwind 4 tokens`.
- **Storybook**: every shared visual primitive gets a Story. Shell components also get one. Pages do not (too data-heavy).
- **MSW**: API mocking for vitest + storybook. Handlers live in `apps/web/src/mocks/handlers.ts`; per-test overrides via `server.use(...)`.
- **Capability default for tests**: assume ZERO unless test explicitly sets CLOUD. ZERO is the production-default surface; the test suite must keep it healthy.
- **Screenshot evidence**: for visual-parity tasks, attach side-by-side screenshot to commit body (`docs/internal/screenshots/m1b-<screen>.png` — these are internal, not shipped).

### 2.2 axios wrapper

- [ ] Create `apps/web/src/lib/api-client.ts`:
  ```ts
  import axios, { type AxiosError, type AxiosInstance } from "axios";
  import type { paths } from "@/lib/api-types";

  export class ApiError extends Error {
    constructor(public status: number, public code: string, message: string, public retryable: boolean) {
      super(message);
    }
  }

  function createClient(): AxiosInstance {
    const inst = axios.create({ baseURL: "/api/v1", withCredentials: true, timeout: 30_000 });
    inst.interceptors.response.use(
      (r) => r,
      (err: AxiosError<{ code?: string; message?: string }>) => {
        const status = err.response?.status ?? 0;
        const code = err.response?.data?.code ?? "UNKNOWN";
        const message = err.response?.data?.message ?? err.message;
        if (status === 401) {
          window.location.assign("/login?next=" + encodeURIComponent(window.location.pathname));
        }
        const retryable = status === 0 || status >= 500;
        throw new ApiError(status, code, message, retryable);
      },
    );
    return inst;
  }

  export const api = createClient();
  export type Paths = paths;
  ```

## Task 3 — Capability store + Gated component

Goal: app-boot capability fetch into Zustand; `<Gated>` wrapper renders children iff feature flag is enabled. ZERO tier hides AI surfaces.

### 3.1 Zustand store

- [ ] Create `apps/web/src/stores/use-capabilities.ts`:
  ```ts
  import { create } from "zustand";
  import { api } from "@/lib/api-client";

  export type Tier = "ZERO" | "LOCAL" | "CLOUD";
  export type FeatureKey =
    | "ai_generation"
    | "ai_diagnosis"
    | "ai_translation"
    | "ai_conversation"
    | "ai_panel"
    | "autonomy_assist"
    | "autonomy_semi_auto"
    | "autonomy_auto"
    | "embeddings_semantic";

  export type Capabilities = {
    tier: Tier;
    autonomy: "manual" | "assist" | "semi_auto" | "auto";
    llm_provider: string | null;
    model: string | null;
    features: Record<FeatureKey, boolean>;
  };

  type State = {
    capabilities: Capabilities | null;
    loading: boolean;
    error: string | null;
    fetch: () => Promise<void>;
    setCapabilities: (c: Capabilities) => void;
  };

  export const useCapabilities = create<State>((set) => ({
    capabilities: null,
    loading: true,
    error: null,
    fetch: async () => {
      set({ loading: true, error: null });
      try {
        const res = await api.get<Capabilities>("/capabilities");
        set({ capabilities: res.data, loading: false });
      } catch (e) {
        set({ error: (e as Error).message, loading: false });
      }
    },
    setCapabilities: (c) => set({ capabilities: c }),
  }));
  ```

### 3.2 hook

- [ ] Create `apps/web/src/hooks/use-feature-enabled.ts`:
  ```ts
  import { useCapabilities, type FeatureKey } from "@/stores/use-capabilities";

  export function useFeatureEnabled(feature: FeatureKey): boolean {
    return useCapabilities((s) => Boolean(s.capabilities?.features?.[feature]));
  }
  ```

### 3.3 Gated component

- [ ] Create `apps/web/src/components/gating/Gated.tsx`:
  ```tsx
  import type { ReactNode } from "react";
  import { useFeatureEnabled } from "@/hooks/use-feature-enabled";
  import type { FeatureKey } from "@/stores/use-capabilities";

  type Props = { feature: FeatureKey; fallback?: ReactNode; children: ReactNode };

  export function Gated({ feature, fallback = null, children }: Props) {
    const enabled = useFeatureEnabled(feature);
    if (!enabled) return <>{fallback}</>;
    return <>{children}</>;
  }
  ```

## Task 8 — WebSocket placeholder client

Goal: scaffold a typed WS client used in M1c for run log streaming + capability change events. M1b only validates reconnect + subscribe API via unit test — not consumed yet by any screen.

### 8.1 Implementation

- [ ] Create `apps/web/src/lib/ws-client.ts`:
  ```ts
  type Listener = (payload: unknown) => void;
  type Topic = string;

  export class WsClient {
    private socket: WebSocket | null = null;
    private listeners = new Map<Topic, Set<Listener>>();
    private subscribed = new Set<Topic>();
    private reconnectAttempts = 0;
    private url: string;

    constructor(url: string) { this.url = url; }

    connect() {
      this.socket = new WebSocket(this.url);
      this.socket.onopen = () => {
        this.reconnectAttempts = 0;
        for (const t of this.subscribed) this.socket?.send(JSON.stringify({ type: "subscribe", topic: t }));
      };
      this.socket.onmessage = (ev) => {
        try {
          const msg = JSON.parse(ev.data) as { topic: string; event: string; payload: unknown };
          const set = this.listeners.get(msg.topic);
          set?.forEach((cb) => cb({ event: msg.event, payload: msg.payload }));
        } catch { /* ignore malformed */ }
      };
      this.socket.onclose = () => this.scheduleReconnect();
      this.socket.onerror = () => this.socket?.close();
    }

    private scheduleReconnect() {
      const backoff = Math.min(30_000, 500 * 2 ** this.reconnectAttempts);
      this.reconnectAttempts += 1;
      setTimeout(() => this.connect(), backoff);
    }

    subscribe(topic: Topic, cb: Listener): () => void {
      this.subscribed.add(topic);
      let set = this.listeners.get(topic);
      if (!set) { set = new Set(); this.listeners.set(topic, set); }
      set.add(cb);
      if (this.socket?.readyState === WebSocket.OPEN) {
        this.socket.send(JSON.stringify({ type: "subscribe", topic }));
      }
      return () => {
        set!.delete(cb);
        if (set!.size === 0) {
          this.subscribed.delete(topic);
          this.listeners.delete(topic);
          if (this.socket?.readyState === WebSocket.OPEN) {
            this.socket.send(JSON.stringify({ type: "unsubscribe", topic }));
          }
        }
      };
    }

    close() { this.socket?.close(); }
  }

  export const wsClient = new WsClient(`${window.location.protocol === "https:" ? "wss" : "ws"}://${window.location.host}/ws`);
  ```
